# ④ UI設計

## 1. 画面構成 / ナビゲーション

スマホはボトムタブ、PC(md以上)はサイドナビに切り替える(shadcn/uiの`Sheet`/`NavigationMenu`を利用)。

```
┌─────────────────────────┐
│        Header            │  ← ページタイトル、ダークモード切替、通知(将来)
├─────────────────────────┤
│                           │
│        Content            │
│                           │
├─────────────────────────┤
│ 🏠  🥬  🍳  🛒  ⚙️      │  ← BottomNav (スマホのみ、fixed)
│ダッシュ 在庫 レシピ 買い物 設定│
└─────────────────────────┘
```

- タブは5つ固定: ダッシュボード / 在庫一覧 / レシピ提案 / 買い物リスト / 設定
- 献立提案はレシピタブ内のサブタブとして統合(タブを増やしすぎない)
- PC幅(`md:`以上)ではサイドバー化 + コンテンツ幅を`max-w-4xl`程度に制限し余白を活かす

---

## 2. デザイントークン

| 項目 | 方針 |
|---|---|
| ベースカラー | shadcn/ui標準の`zinc`ベース。強調色は食欲を刺激する暖色系(`orange-500`系)をブランドカラーとしてボタン/アクセントに限定使用 |
| ダークモード | `next-themes`で`class`戦略。`prefers-color-scheme`追従 + 手動切替トグルを設定画面に配置 |
| 角丸 | `rounded-2xl`を基調（Apple風の柔らかさ） |
| 余白 | セクション間`space-y-6`〜`8`、カード内`p-4`〜`6`。スマホでも詰め込みすぎない |
| フォント | システムフォント(`font-sans` = Inter代替の可変フォント)。数字(賞味期限・数量)は`tabular-nums` |
| アイコン | Lucide、線幅は標準(strokeWidth=1.75〜2)で統一 |
| シャドウ | 控えめ(`shadow-sm`)、カード境界は`border`メインで表現(フラットに近いApple風) |

## 3. 主要画面

### 3.1 ダッシュボード (`/`)
- 「期限切れ」「期限間近(3日以内)」「登録食材数」を横並びStatカード(スマホは2列グリッド)
- 「今日使うべき食材」カード → AIが賞味期限の近い食材からおすすめレシピを1件ハイライト表示(要件④の「豆腐が明日期限なので麻婆豆腐」を実現)
- 下部に「最近のAIおすすめ料理」カルーセル

### 3.2 在庫一覧 (`/ingredients`)
- 上部: 検索バー + フィルタ(カテゴリチップ) + ソート(期限順/名前順/残り少ない順)をドロップダウン
- リスト項目(`ingredient-card`): 食材名、カテゴリアイコン、数量+単位、賞味期限バッジ(色分け: 期限切れ=赤、3日以内=橙、それ以外=グレー)
- 各カードに **ワンタップ増減ボタン**(`-` / `+`、長押しで直接入力ダイアログ)
- 右下にFAB(Floating Action Button)で食材追加ダイアログを開く
- 食材追加/編集フォーム: 名前・数量・単位・カテゴリ(セレクト)・保存場所(セレクト)・賞味期限(DatePicker)・メモ。React Hook Form + Zod

### 3.3 レシピ提案 (`/recipes`)
- タブ: 「作れる料理」「あと1品で作れる」「献立」「お気に入り」「履歴」
- 「作れる料理」: 現在庫を元にAIが5件カード表示。各カードに難易度バッジ・調理時間・材料アイコン。タップで詳細ダイアログ(手順・必要材料・お気に入りボタン)
- 「あと1品で作れる」: 不足食材を1つ入力/選択 → 候補レシピ一覧
- 「献立」: 今日の夕飯/明日の朝/今週の献立をセグメントコントロールで切替
- ローディング中はshadcnの`Skeleton`でカード形状のプレースホルダ表示(AI応答待ちのUX配慮)

### 3.4 買い物リスト (`/shopping-list`)
- チェックリストUI(shadcn `Checkbox`)。チェック済みは取り消し線+グレーアウトし下部に集約
- 「AIが提案した不足食材を追加」ボタン → レシピ提案から生成された不足食材を一括インポート
- 手動追加は下部の常設入力欄(Enterで即追加、モバイルのフリック入力を邪魔しない)
- 世帯内でRealtime同期(相手が追加/チェックすると即反映、軽いトースト通知)

### 3.5 設定 (`/settings`)
- 世帯設定: 招待URL表示 + コピー ボタン + 再発行(確認ダイアログ付き)
- プロフィール: 表示名、アレルギー/苦手食材(タグ入力)、ダイエット方針(セレクト: 高タンパク/低脂質/指定なし)
- 表示設定: ダークモード切替
- (将来)LINE通知連携、通知タイミング設定

## 4. コンポーネント設計方針(shadcn/ui)

| 用途 | shadcn/uiコンポーネント |
|---|---|
| カード全般 | `Card` |
| フォーム | `Form` (react-hook-form連携) + `Input` / `Select` / `Textarea` / `Popover`+`Calendar`(日付) |
| ダイアログ | `Dialog`(PC) / モバイルでは`Drawer`(vaul)に自動切替 |
| タブ | `Tabs` |
| チェックリスト | `Checkbox` |
| バッジ | `Badge`(賞味期限の色分けにvariant追加) |
| トースト通知 | `Sonner` |
| ローディング | `Skeleton` |
| ナビ | `NavigationMenu`(PC) / 自前`BottomNav`(モバイル、shadcn非対応のため独自実装) |

モバイルでのダイアログは画面が狭いため、`Dialog`ではなく下から出る`Drawer`(vaul使用、shadcn/uiの`Drawer`コンポーネント)を優先し、片手操作を重視する。

---

## 5. アクセシビリティ / レスポンシブ

- タップ領域は最小44x44px確保(特にワンタップ増減ボタン)
- カラーコントラストはWCAG AA準拠(赤バッジ等はダークモードでも視認性を個別検証)
- ブレークポイント: `sm`未満=1カラム、`sm`〜`md`=2カラムグリッド、`lg`以上=3カラム+サイドナビ

---

## 6. この工程での設計レビュー観点

- ✅ タブを5つに絞り、献立提案をレシピ内に統合することで、家庭用アプリとして迷わないナビゲーションにした
- ✅ ワンタップ増減・期限切れ強調・AI提案のローディングスケルトンなど、「毎日使う」ことを前提にしたUX配慮を含めた
- ✅ モバイルはDrawer、PCはDialogという使い分けで、レスポンシブ対応をコンポーネントレベルで吸収
- ⚠️ カラーパレットは初期はzinc+orangeで開始し、実装後に実機(iPhone/Android)で視認性を確認して微調整する
